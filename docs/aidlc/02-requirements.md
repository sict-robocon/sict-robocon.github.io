# Requirements

## Functional

1. The site must include a strong landing section with a short explanation of the Robocon journey.
2. The site must include a chronological timeline for seasons or milestones.
3. The site must include featured build cards for important robots, subsystems, or projects.
4. The site must include lessons or principles learned through competition.
5. The site must include a closing section that supports GitHub publishing and future custom-domain use.

## Content

1. Sample content should be provided so the site is immediately viewable.
2. The sample content must live in a single JavaScript data file for easy replacement.
3. The copy should read like a personal engineering narrative, not a corporate landing page.

## Experience

1. The layout must be responsive without requiring a framework.
2. Motion should be light and meaningful: reveal animations, timeline emphasis, and subtle hover states.
3. Visual hierarchy must be strong enough that a visitor can scan the page in under a minute.

## Technical

1. Use plain HTML, CSS, and JavaScript for maximum portability.
2. Use relative asset paths so the site works in GitHub Pages project repositories.
3. Include a GitHub Actions workflow for Pages deployment.
