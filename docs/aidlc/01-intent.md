# Intent

## Goal

Build a personal, single-page website that tells the history of a Robocon journey in a way that feels memorable, credible, and easy to publish on GitHub Pages.

## Audience

- Teammates and mentors who know the technical context
- Recruiters or judges who need a fast overview
- Friends and visitors who only need the story, not the full engineering detail

## Story Outcome

The site should answer four questions quickly:

1. When did the Robocon journey start?
2. What major seasons, robots, or turning points mattered?
3. What skills and lessons came out of those seasons?
4. Where can the visitor go next to see code, photos, or contact details?

## Delivery Constraints

- Must work as a static site with no backend
- Must publish cleanly to GitHub Pages
- Must be easy to customize by editing one data file
- Must look intentional on both desktop and mobile
- Must support later addition of a custom domain

## Product Direction

The visual language should feel like a mix of race telemetry, workshop notes, and blueprint surfaces rather than a generic portfolio template.
